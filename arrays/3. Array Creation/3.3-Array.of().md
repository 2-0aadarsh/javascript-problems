# 1. Why was `Array.of()` introduced?

Before ES6 developers had only

```js
Array(...)
```

Unfortunately `Array()` behaves differently depending on the number and type of arguments.

Example

```js
Array(5)
```

creates

```text
Sparse array

length : 5
```

while

```js
Array(1,2,3)
```

creates

```js
[1,2,3]
```

This inconsistency confused developers.

So ES6 introduced

```js
Array.of(...)
```

whose behavior never changes.

---

# 2. Mental Model

Think of

```js
Array(...)
```

as

```
Smart Constructor
```

It first decides what you meant.

Think of

```js
Array.of(...)
```

as

```
Simple Factory
```

It never guesses.

Every argument becomes an element.

---

# 3. Engine Decision Tree

When the engine encounters

```js
Array.of(...)
```

there is NO special case.

Decision Tree

```
Array.of(...)

↓

Allocate new Array object

↓

Reference #A

↓

{

    length : 0

}

↓

Insert every argument one by one

↓

Update length after every insertion

↓

Return Reference
```

Notice

Unlike `Array()`

there is NO

```
Single argument?

↓

Number?

↓

Length mode?
```

Everything is treated as data.

# 6. Array.of() vs Array()

Most important interview question.

```js
Array(5)
```

↓

```
Reference

↓

{

length : 5

}
```

Five holes.

No properties.

---

```js
Array.of(5)
```

↓

```
Reference

↓

{

"0":5,

length:1

}
```

One element.

No holes.

---

# 7. Array.of(undefined)

```js
Array.of(undefined)
```

Execution

```
Reference

↓

{

"0" : undefined,

length : 1

}
```

Notice

Property

```
"0"
```

actually exists.

Therefore

```js
0 in arr
```

returns

```js
true
```

---

Compare with

```js
Array(1)
```

Memory

```
Reference

↓

{

length : 1

}
```

No property

```
"0"
```

exists.

Therefore

```js
0 in arr
```

returns

```js
false
```

Huge interview topic.

---

# 8. Array.of() with Arrays

Example

```js
const arr = Array(3);

const result = Array.of(arr);
```

Memory

```
Reference #A

↓

{

length : 3

}
```

---

Result

```
Reference #B

↓

{

"0" : Reference #A,

length : 1

}
```

Notice

`Array.of()` stores

```
Reference #A
```

NOT a copy.

---

# 9. Nested Modification

Example

```js
result[0][1] = 100;
```

Execution

Step 1

```
result

↓

Reference #B
```

Step 2

Read

```
property "0"

↓

Reference #A
```

Step 3

Evaluate

```
Reference #A[1]
```

Property exists?

```
NO
```

Step 4

Create property

```
"1"
```

↓

Assign

```
100
```

Memory

```
Reference #A

↓

{

"1" : 100,

length : 3

}
```

Notice

Properties

```
"0"

"2"
```

still DO NOT exist.

They remain holes.

---

# 10. Important Observation

`Array.of()` never performs deep copy.

When objects or arrays are passed

it stores

```
Reference
```

not

```
Copied Object
```

Example

```js
Array.of(obj)
```

Memory

```
Reference

↓

{

"0" : ReferenceToObject

}
```

Both variables point to the same object.

Changing one changes the other.

---

```
Magic
```

```

↓

↓

Insert argument 2

↓

```

Exactly like repeated `push()` operations.

---

# 12. Comparison Table

| Feature                 | Array()           | Array.of()           |
| ----------------------- | ----------------- | -------------------- |
| Multiple arguments      | Stores elements   | Stores elements      |
| Single numeric argument | Treated as length | Stored as element    |
| Creates holes           | Yes               | Never                |
| Stores references       | Yes               | Yes                  |
| Deep copy               | No                | No                   |
| Shallow copy            | N/A               | References preserved |
