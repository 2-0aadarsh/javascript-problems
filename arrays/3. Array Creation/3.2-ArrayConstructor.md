# 1. Array() vs new Array()

Both behave exactly the same.

```js
Array(5)

new Array(5)
```

Both create a brand new Array object.

Unlike most constructors in JavaScript, Array can be invoked without `new`.

Internally both create an Array instance.

---

# 2. Array Constructor Decision Tree

Whenever the engine sees

```js
Array(...)
```

it does NOT immediately create an array.

Instead it first decides WHICH constructor path to execute.

Decision Tree

```

Array(...)
↓
Exactly one argument?

├───────────────┐
│               │
No              Yes
│               │
↓               ↓
Store every     Is it a Number?
argument        │
                ├─────────────┐
                │             │
               No            Yes
                │             │
                ↓             ↓
        Store as element   Is it a valid array length?
                                 │
                     ├───────────┴───────────┐
                     │                       │
                    Yes                     No
                     │                       │
                     ↓                       ↓
             Create holes            RangeError

```

# 3. Multiple Arguments

```js
Array(1,2,3)
```

Engine:

```

More than one argument

↓

Store every argument

```

Result

```
Reference #A

↓

{

    "0" : 1,

    "1" : 2,

    "2" : 3,

    length : 3

}
```

Equivalent to

```js
[1,2,3]
```

---

# 4. Single Non-number Argument

Examples

```js
Array("Hello")

Array(false)

Array({})

Array([])
```

Engine

```

Exactly one argument

↓

NOT a Number

↓

Store as element 0

```

Example

```
Reference #A

↓

{

    "0" : "Hello",

    length : 1

}
```

---

# 5. Single Number Argument

Example

```js
Array(5)
```

Engine

```

Exactly one argument

↓

Number

↓

Valid Array Length

↓

Create Sparse Array

```

Result

```
Reference #A

↓

{

    length : 5

}
```

Notice

There are NO properties

```
"0"

"1"

"2"

...
```

Only

```
length : 5
```

exists.

---

# 6. Valid Array Length

A valid array length must satisfy ALL conditions:

- Number
- Integer
- >= 0
- <= 2³² − 1

Examples

Valid

```js
Array(0)

Array(1)

Array(100)

Array(4294967295)
```

Invalid

```js
Array(-1)

Array(3.5)

Array(4294967296)
```

Invalid lengths throw

```
RangeError
```

---

# 7. Why Holes Exist

Imagine

```js
Array(1_000_000_000)
```

If the engine immediately created

```
{

"0":undefined,

"1":undefined,

...

"999999999":undefined

}
```

it would

- Perform one billion property insertions
- Consume enormous memory
- Waste CPU time
- Potentially crash the program

Instead

the engine stores only

```
{

length : 1000000000

}
```

This design is memory-efficient.

---

# 8. Sparse Arrays

Arrays created with

```js
Array(length)
```

are called sparse arrays.

Example

```
Reference #A

↓

{

length : 5

}
```

Properties

```
0 ❌

1 ❌

2 ❌

3 ❌

4 ❌
```

Only length exists.

---

# 9. Hole vs Undefined

Huge interview topic.

Hole

```
Property does NOT exist.
```

Undefined

```
Property EXISTS

↓

Value = undefined
```

These are different.

Example

```js
const arr = Array(3);

0 in arr
```

Result

```
false
```

because property "0" does not exist.

But

```js
arr[0]
```

returns

```
undefined
```

because accessing a missing property yields undefined.

---

# 10. Lazy Allocation

The engine delays creating properties until necessary.

Example

```js
const arr = Array(1000);

arr[5] = 100;
```

Memory becomes

```
Reference #A

↓

{

"5" : 100,

length : 1000

}
```

Properties

```
0

1

2

3

4

6

...
```

still DO NOT exist.

---

# 11. Mental Model

Never imagine

```
Array(5)

↓

{

"0":undefined,

"1":undefined,

"2":undefined,

...

}
```

Correct mental model

```
Reference #A

↓

(Array Object)

↓

{

length : 5

}
```

Everything else is a hole.
